The AI Tools Stack I Recommend to Every IT Leader

A curated stack of AI tools that actually work for IT operations — from chatbots to orchestrators to local models.

There are thousands of AI tools in the market. Most of them are not worth your time. After building, testing, and categorizing dozens of them — including the full compendium on this site — here's the short list I recommend to every IT leader.

Layer 1: The Everyday Workhorses

These are tools your team can start using today with zero setup:

  • ChatGPT or Claude — For drafting, summarizing, brainstorming, and research. Pick one and make it the team standard. Claude is better for long documents, ChatGPT for broader use cases.
  • Perplexity — AI-powered search with citations. Replace your "let me Google that" habit. Great for technical research and root cause analysis.
  • Grammarly AI — Professional communication is underrated. This catches tone, clarity, and grammar issues in client-facing emails and documents.

Tactic: Pick one tool from this layer and run a one-week team experiment (see my mentoring framework post for how).

Layer 2: Coding & Automation

These tools are for teams that write scripts, manage infrastructure, or build internal tools:

  • GitHub Copilot — Code completion that works. It's especially good for writing boilerplate, tests, and scripts for IT automation tasks.
  • Cursor — An AI-first code editor. It understands your entire codebase, not just the file you're in. Great for complex debugging.

Layer 3: Orchestration & Agentic Workflows

When you need AI to do more than answer questions — when it needs to act:

  • LangChain or LlamaIndex — For building custom AI workflows that connect to your data, run APIs, and make decisions.
  • Flowise — A no-code UI for LangChain. If your team isn't coding but wants to build AI workflows, start here.
  • CrewAI — For orchestrating multiple AI agents that work together on complex tasks. Think of it as "team of AI assistants."

Advice: Don't jump into this layer until you've gotten value from Layer 1. Orchestration is powerful, but it's also complex. Walk before you run.

Layer 4: Local & Private AI

For teams with data privacy requirements or a desire to avoid SaaS dependency:

  • Ollama — Run Llama, Mistral, and other models locally on a laptop or server. Dead simple to set up.
  • LM Studio — A desktop GUI for running local models. Great for team members who aren't comfortable with command line.
  • LocalAI — A drop-in replacement for OpenAI's API that runs locally. If you've built workflows around OpenAI but want to move them on-premise, this is your tool.

The Key Insight

Most teams I work with get stuck because they try to adopt too many tools at once. They hear about a new tool every week and feel like they're falling behind. But the teams that actually see results do the opposite: they pick one tool, use it consistently for a month, and only then add another.

Start with Layer 1. One tool. One workflow. One week. That's the playbook.
